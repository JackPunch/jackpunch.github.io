### 1. 当在浏览器中输入 Google.com 并且按下回车之后发生了什么？

1. **URL 解析**：首先判断地址栏中是否为URL，如果不是，转到搜索引擎。然后对URL中的非法字符进行转义。
2. **缓存判断**：检查本地缓存中是否存在URL对应的资源，如果有，直接返回缓存中的资源。
3. **DNS 解析**：
    1. 从浏览器和计算机缓存中查找域名对应的 IP 地址，如果有，直接使用；否则进行下一步。
    2. 向本地 DNS 服务器**递归查询**，如果本地服务器中存在缓存，直接返回 IP 地址。
    3. 本地 DNS 服务器发起**迭代查询**：先向根服务器查询顶级域名服务器的地址，再向顶级域名服务器查询权威域名服务器的位置，最后向权威域名服务器查询目标主机的 IP 地址。
4. **获取 MAC 地址**：判断是否在同一子网。将目标 IP 与子网掩码相与，得到目标网络号；将本机 IP 与子网掩码相与，得到本机网络号。如果两者相同，说明在同一子网内。
    - 若在同一子网，先检查本地 ARP 缓存表；若无缓存或缓存过期，发出 ARP 请求广播；目标主机单播响应，报告自己的 MAC 地址。
    - 若不在同一子网，转为获取网关的 MAC 地址，后续请求由网关代发。
5. **TCP 三次握手**：
    1. 客户端发送 SYN 包（同步包）
    2. 服务器收到 SYN 包，返回 SYN-ACK 包（同步-确认包）
    3. 客户端收到 SYN-ACK 包，返回 ACK包（确认包）
6. **TLS 四次握手（HTTPS 加密）**：
    1. 客户端发送 Client Hello 包，包含支持的加密算法和随机数
    2. 服务器返回 Server Hello + Certificate + Server Hello Done 包，包含选择的加密算法、数字证书和另一个随机数
    3. 客户端生成第三个随机数，并用数字证书做公钥加密，同时用三个随机数生成对称加密的密钥。发送数据有 Client Key Exchange（加密后的随机数）, Change Cipher Spec（表明变更密码规格协议）, Encrypted Handshake Message（加密后的握手消息）
    4. 服务器收到加密后的第三个随机数，用私钥解密，随后用三个随机数生成对称加密的密钥。发送 Change Cipher Spec 和 Encrypted Handshake Message 包，表示握手完成。
7. **发送 HTTP 请求并收到响应**
8. **页面渲染**：根据 HTML 文件构建 DOM 树，根据 CSS 文件构建 CSSOM 树。根据 DOM 树和 CSSOM 树构建渲染树，进行布局和绘制。
9. **TCP 四次挥手**：
    1. 客户端发送 FIN 包（结束包）
    2. 服务器收到 FIN 包，返回 ACK 包（确认包）
    3. 服务器发送 FIN 包
    4. 客户端收到 FIN 包，返回 ACK 包
